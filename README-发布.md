# 第七次敲门：发布说明

这个项目已经是标准静态网站。

可直接发布的文件：

- `index.html`
- `styles.css`
- `script.js`
- `背景图片/`
- `立绘/`
- `音乐/`

发布时只要保持目录结构不变即可。

## 方案 1：最快分享

适合先发给朋友试玩，不追求中国大陆稳定访问。

可用平台：

- Cloudflare Pages
- GitHub Pages

### Cloudflare Pages

1. 注册并登录 Cloudflare。
2. 进入 `Workers & Pages`。
3. 新建 Pages 项目。
4. 直接上传整个项目目录，或连接 Git 仓库。
5. 构建设置留空或使用：

```txt
Build command: 留空
Build output directory: /
```

6. 发布后得到一个 `*.pages.dev` 域名。

说明：

- 这个项目没有构建步骤，纯静态上传即可。
- 适合快速分享。
- 但中国大陆访问体验不保证稳定。

## 方案 2：优先保证中国大陆可打开

这是更推荐的正式发布方式。

可用平台：

- 阿里云 OSS 静态网站
- 腾讯云 COS 静态网站

### 阿里云 OSS

1. 注册域名。
2. 创建 OSS Bucket。
3. 开启静态网站托管。
4. 上传整个项目目录中的所有文件。
5. 设置首页为 `index.html`。
6. 绑定你自己的域名。
7. 将域名解析到 OSS。

注意：

- 如果 Bucket 在中国大陆地域，正式对外访问通常需要域名备案。
- 为了让浏览器正常访问页面，建议绑定自定义域名，不要只用 Bucket 域名。

### 腾讯云 COS

1. 注册域名。
2. 创建 COS Bucket。
3. 开启静态网站功能。
4. 上传整个项目目录中的所有文件。
5. 设置首页为 `index.html`。
6. 绑定自定义域名。
7. 如有需要，开启 CDN 加速。

注意：

- 中国大陆访问使用自定义域名更合适。
- 如果面向中国大陆正式提供访问，通常也需要备案。

## 我建议你这样发

如果你只是想先发给朋友看：

- 先上 Cloudflare Pages

如果你想让中国大陆也尽量稳定打开：

- 买一个域名
- 用阿里云 OSS 或腾讯云 COS
- 绑定自定义域名
- 做备案

## 当前项目是否还需要改代码

不需要。

当前版本已经满足静态网站发布要求：

- 没有后端依赖
- 没有数据库依赖
- 没有构建依赖
- 所有资源都用相对路径引用

也就是说，只要上传文件，就能运行。
